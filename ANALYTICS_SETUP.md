# Подсистема аналитики и статистики

## Что было создано

✅ **Django приложение "analytics"** - для аналитики и статистики
✅ **Dashboard с графиками:**
   - График продаж по дням (линейный график)
   - Топ-10 популярных товаров (столбчатая диаграмма)
   - Распределение заказов по статусам (круговая диаграмма)
✅ **Метрики:**
   - Общая сумма продаж за период
   - Количество заказов за период
   - Средний чек
   - Количество новых пользователей за период
✅ **Фильтры по датам** - выбор периода для анализа
✅ **Экспорт в CSV** - выгрузка данных статистики
✅ **Интеграция с RBAC** - доступ только для Manager и Admin

## Как использовать

### Доступ к dashboard

1. Войдите как менеджер или администратор
2. В главном меню появится ссылка "Аналитика"
3. Или перейдите напрямую: `/analytics/dashboard/`

### Фильтрация по датам

1. Выберите дату начала периода
2. Выберите дату окончания периода
3. Нажмите "Применить фильтр"
4. Графики и метрики обновятся

### Экспорт данных

1. Выберите период (даты)
2. Нажмите кнопку "Экспорт CSV"
3. Файл будет скачан с данными статистики

## Графики

### 1. График продаж по дням (линейный)

- **Тип:** Линейный график (Chart.js)
- **Данные:** Сумма заказов по дням за выбранный период
- **Источник:** `Order.total_amount`
- **Фильтр:** Исключаются отмененные заказы

### 2. Топ-10 популярных товаров (столбчатая диаграмма)

- **Тип:** Столбчатая диаграмма (Chart.js)
- **Данные:** Количество проданных единиц каждого товара
- **Источник:** `OrderItem.quantity`
- **Группировка:** По `Product`

### 3. Распределение заказов по статусам (круговая диаграмма)

- **Тип:** Круговая диаграмма (doughnut chart)
- **Данные:** Процентное соотношение заказов в разных статусах
- **Источник:** `Order.status`
- **Статусы:** pending, confirmed, preparing, shipping, delivered, cancelled

## Метрики

### Общая сумма продаж
- Сумма всех заказов за период (исключая отмененные)
- Формат: рубли (₽)

### Количество заказов
- Общее количество заказов за период (исключая отмененные)

### Средний чек
- Средняя сумма заказа за период
- Формат: рубли (₽)

### Новых пользователей
- Количество зарегистрированных пользователей за период

## Права доступа

- **Manager** - полный доступ к аналитике
- **Admin** - полный доступ к аналитике
- **Customer** - доступ запрещен (декоратор `@manager_required`)

## Технические детали

### Используемые библиотеки

- **Chart.js 4.4.0** - для визуализации графиков
- **Bootstrap 5** - для стилизации (уже используется в проекте)
- **Django ORM** - для запросов к БД

### Оптимизация запросов

- Используются агрегатные функции Django (`Sum`, `Count`, `Avg`)
- Группировка данных на уровне БД
- Индексы на полях `created_at` для быстрого поиска

### Обработка пустых данных

Если за выбранный период нет данных:
- Показывается сообщение: "За выбранный период данных нет"
- Графики не отображаются
- Метрики показывают нулевые значения

## Примеры использования

### Получение статистики программно

```python
from analytics.utils import (
    get_sales_by_date,
    get_top_products,
    get_orders_by_status,
    get_sales_metrics
)
from datetime import datetime, timedelta
from django.utils import timezone

# Статистика за последние 30 дней
start_date = timezone.now() - timedelta(days=30)
end_date = timezone.now()

# Продажи по дням
sales = get_sales_by_date(start_date, end_date)

# Топ товаров
top_products = get_top_products(10, start_date, end_date)

# Распределение по статусам
statuses, total = get_orders_by_status(start_date, end_date)

# Метрики
metrics = get_sales_metrics(start_date, end_date)
```

## URL маршруты

- `/analytics/dashboard/` - главная страница аналитики
- `/analytics/export/` - экспорт данных в CSV (с параметрами `start_date` и `end_date`)

## Интеграция с меню

Ссылка "Аналитика" автоматически появляется в главном меню для пользователей с ролью Manager или Admin.

## Примечания

- Графики используют Chart.js через CDN
- Данные передаются в JSON формате для безопасности
- CSV экспорт использует кодировку UTF-8
- Все даты обрабатываются с учетом timezone Django
