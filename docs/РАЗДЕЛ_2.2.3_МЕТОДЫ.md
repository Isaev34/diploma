# 2.2.3. Методы

При разработке веб-сервиса применялся стек технологий: Django 4.2 на Python для серверной части и Django REST Framework (DRF) для REST API. Сборка и развёртывание выполняются стандартным инструментарием Django (manage.py, WSGI/ASGI); маршрутизация реализована через URL-конфигурацию и DRF routers. Работа с базой данных — через Django ORM (SQLite в разработке, PostgreSQL в продакшене).

Безопасность обеспечивается djangorestframework-simplejwt (JWT-токены) и встроенными механизмами Django (PBKDF2 для паролей). Валидация данных выполняется через DRF Serializers. Документация API — drf-spectacular (OpenAPI/Swagger), фильтрация — django-filter, администрирование — django-jazzmin, валидация адресов доставки — библиотека dadata.

Приложение реализовано с опорой на парадигмы объектно-ориентированного программирования: инкапсуляция, наследование, полиморфизм и абстракция. Модели наследуются от `models.Model` Django — это абстрагирует работу с БД. Связи между сущностями оформлены через `ForeignKey` и `related_name`, что инкапсулирует логику доступа к связанным данным. Модульная структура (приложения users, catalog, cart, api, analytics, shifts; модели, сериализаторы, представления) поддерживает соблюдение принципов ООП.

## Примеры парадигм ООП в проекте (где делать скриншоты)

### Рисунок 16 — Пример абстракции

**Файл:** `catalog/models.py`  
**Строки:** 124–131 (свойство `final_price` у модели `Product`).

Откройте `catalog/models.py`, найдите класс `Product` и метод `final_price` (помечен декоратором `@property`). Детали расчёта цены со скидкой скрыты за обращением `product.final_price`.

---

### Рисунок 17 — Пример инкапсуляции

**Файл:** `catalog/models.py`  
**Строки:** 148–158 — метод `decrease_stock`, 160–163 — метод `increase_stock` (модель `Product`).

Либо:

**Файл:** `users/models.py`  
**Строки:** 43–56 — методы `add_bonus_points` и `spend_bonus_points` (модель `User`).

Состояние изменяется только через эти методы. Для скриншота достаточно одного файла: либо `catalog/models.py` (склад), либо `users/models.py` (бонусы).
---

### Рисунок 18 — Пример наследования

**Файл:** `users/models.py`  
**Строки:** 1–41 (импорт `AbstractUser` и класс `User(AbstractUser)` с полями `phone`, `bonus_points` и т.д.).

Откройте `users/models.py` с начала файла. На скриншоте должны быть видны строка `class User(AbstractUser):` и несколько добавленных полей.

---

### Рисунок 19 — Пример полиморфизма

**Файл:** `api/views.py`  
**Строки:** 36–55 — классы `CategoryViewSet` и `ProductViewSet` (оба наследуют `viewsets.ModelViewSet`).

Откройте `api/views.py`, найдите подряд идущие `CategoryViewSet` и `ProductViewSet`. Один интерфейс `ModelViewSet`, разные модели и сериализаторы.

**Альтернатива для скриншота:** `api/serializers.py` — классы `CategorySerializer` и `ProductSerializer` (наследование от `serializers.ModelSerializer`).
