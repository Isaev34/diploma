# 3. ТЕХНОЛОГИЧЕСКАЯ ЧАСТЬ

## 3.1. Инструментальные средства

В рамках проекта для хранения и обработки данных была выбрана **PostgreSQL** — стабильная и высокопроизводительная СУБД с открытым исходным кодом. Она обеспечивает надёжность, гибкость масштабирования и широкий набор возможностей для работы с данными, а также хорошо подходит для веб-приложений на Django.

Разработка велась в среде **Visual Studio Code**. Эта среда предоставляет удобные условия для программирования, встроенную отладку, поддержку Python и расширений для Django и REST API.

Серверная часть проекта реализована на **Python** с использованием фреймворка **Django 4.2** и **Django REST Framework (DRF)**. Такой стек позволил создать веб-сервис с единой кодовой базой: сайт с каталогом, корзиной и заказами (Django-шаблоны), REST API для мобильных приложений сборщика и курьера, панель администрирования (Jazzmin). DRF обеспечивает типизацию запросов и ответов, аутентификацию по JWT и автоматическую документацию API (drf-spectacular, Swagger UI).

Клиентская часть веб-сайта (страницы каталога, корзины, профиля, оформления заказа) выполнена на **Django Template Language** с использованием статики и скриптов в браузере. Отдельная клиентская сборка на React не применялась; взаимодействие с сервером реализовано через формы и при необходимости через запросы к API с фронтенда.
