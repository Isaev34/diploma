## Рекомендуемое содержание рисунков (по разделам сайта)

Схему можно разбить на **четыре части** по логическим блокам.

### Рисунок 22 — Общая структура сайта (верхний уровень)

Корневой уровень: основные разделы и их подсистемы.

- **Главная** (`/`) — каталог (catalog)
- **Пользователи** (`/users/`) — авторизация, профиль
- **Корзина и заказы** (`/cart/`)
- **REST API** (`/api/`)
- **Админ-панель** (`/admin/`)
- **Аналитика** (`/analytics/`)
- **Смены** (API: `/api/shifts/`)

Связи: от корня к каждому разделу (блок-схема или дерево).

---

### Рисунок 23 — Раздел «Каталог» (catalog)

Страницы и переходы внутри каталога.

- `/` — главная (home)
- `/categories/` — список категорий
- `/categories/<slug>/` — товары категории
- `/products/` — список товаров
- `/products/<slug>/` — карточка товара

Связи: главная → категории → категория → товары → товар.

---

### Рисунок 24 — Разделы «Пользователи» и «Корзина / Заказы»

**Пользователи (`/users/`):**

- `/users/login/` — вход
- `/users/logout/` — выход
- `/users/signup/` — регистрация
- `/users/profile/` — профиль
- `/users/profile/edit/` — редактирование профиля
- `/users/cards/`, `/users/cards/add/` — карты
- `/users/addresses/`, `/users/addresses/add/` — адреса

**Корзина и заказы (`/cart/`):**

- `/cart/` — корзина
- `/cart/add/<product_id>/` — добавить в корзину
- `/cart/update/<item_id>/`, `/cart/remove/<item_id>/` — изменить / удалить
- `/cart/checkout/` — оформление заказа
- `/cart/orders/` — список заказов
- `/cart/orders/<id>/` — страница заказа

Связи: логин/регистрация → профиль → карты, адреса; каталог → корзина → оформление → заказы.

---

### Рисунок 25 — API и служебные разделы

**REST API (`/api/`):**

- `GET/POST /api/categories/`, `products/`, `orders/`, `cart/`, `cart-items/`
- `GET/POST/PUT /api/users/`, `addresses/`
- `GET/POST /api/addresses/suggest/`, `product-suggest/`, `delivery-fee-estimate/`
- `GET/POST /api/courier/orders/` — заказы курьера
- `GET/POST /api/picker/orders/` — заказы сборщика
- `POST /api/auth/token/`, `auth/token/refresh/` — JWT
- `GET /api/docs/` — Swagger UI
- `GET /api/shifts/` — смены (start, end, stats, history, support)

**Другие разделы:**

- `/admin/` — панель администратора (Django Admin / Jazzmin)
- `/analytics/dashboard/`, `/analytics/export/` — аналитика

Связи: блок «API» с перечислением групп эндпоинтов; отдельно блоки «Админ» и «Аналитика».

---

## Как оформить схемы

- **Вариант 1:** Блок-схема (прямоугольники — разделы/страницы, стрелки — переходы или вложенность).
- **Вариант 2:** Дерево (корень = сайт, ветви = разделы, листья = страницы или ключевые URL).
- **Вариант 3:** Несколько схем в draw.io / Miro / PowerPoint по одному рисунку на раздел.

Подписи к рисункам оставить как в тексте: «Рисунок 22 — Структурная схема» и т.д. (при необходимости уточнить: «Рисунок 22 — Структурная схема WEB-сайта (общая структура)» и т.п.).
