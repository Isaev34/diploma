2.2. Внешняя спецификация

2.2.1. Описание задачи

Разработать веб-сайт, обеспечивающий продажу продуктов питания с доставкой.

Данное программное решение должно состоять из реляционной базы данных под управлением СУБД PostgreSQL и информационной системы, реализованной на веб-платформе.

Веб-ресурс должен иметь возможность отображения различной информации, хранящейся в базе данных, с помощью технологии API.

На этапе изучения темы были определены основные роли: неавторизованный покупатель, покупатель (клиент), менеджер, администратор, сборщик заказов и курьер. Роли сборщика и курьера реализованы в системе и взаимодействуют с платформой через мобильные приложения (API); в данной спецификации детально описаны возможности ролей, использующих веб-интерфейс.

Рисунок 1 - Диаграмма прецедентов.

На основе спроектированной диаграммы прецедентов можно выделить возможности каждой роли.

Возможности пользователей по ролям

Пользователь с ролью «Неавторизованный покупатель» обладает следующими возможностями:
- Регистрация в качестве клиента;
- Авторизация в системе;
- Просмотр главной страницы;
- Вывод списка товаров в каталоге (продукты питания по категориям);
- Просмотр страницы товара с подробной информацией;
- Поиск товаров по названию и описанию;
- Фильтрация товаров по категории;
- Сортировка товаров (по цене, по названию, по дате добавления);

Пользователь с ролью «Покупатель» (авторизованный покупатель) обладает следующими возможностями:
- Все возможности неавторизованного пользователя;
- Добавление товаров в корзину;
- Изменение количества товара при добавлении в корзину и в корзине;
- Вывод списка товаров в корзине;
- Удаление товаров из корзины;
- Оформление заказа с указанием адреса доставки, способа оплаты и комментариев для курьера и сборщика;
- Списание бонусных баллов при оформлении заказа;
- Просмотр стоимости доставки по введённому адресу (расчёт по расстоянию);
- Сохранение адреса доставки в профиле;
- Добавление и управление привязанными картами (для оплаты онлайн);
- Вывод списка истории заказов;
- Просмотр детальной информации о заказе (статус, позиции, адрес);
- Просмотр информации о своём профиле;
- Изменение личных данных профиля (email, телефон);
- Управление адресами доставки в профиле (добавление, изменение);
- Выход из аккаунта;

Пользователь с ролью «Менеджер» обладает следующими возможностями:
- Просмотр статистики по продажам и доходам (дашборд аналитики за выбранный период);
- Вывод списка заказов пользователей;
- Изменение статусов заказов (в ожидании, в сборке, собран, доставляется, доставлен, отменён);
- Назначение сборщика на заказ;
- Вывод списка товаров;
- Добавление товаров (название, описание, категория, штрихкод, цена, скидка, изображение, остаток на складе);
- Изменение товаров;
- Удаление товаров;
- Загрузка изображений для товаров;
- Вывод списка категорий;
- Добавление категорий;
- Изменение категорий;
- Удаление категорий;

Пользователь с ролью «Администратор» обладает следующими возможностями:
- Вывод списка пользователей;
- Добавление пользователей с любой ролью (покупатель, менеджер, администратор, курьер, сборщик);
- Изменение пользователей (имя пользователя, имя, фамилия, email, телефон, пароль, роль, активность);
- Удаление пользователей;
- Вывод списка логов активности пользователей;
- Просмотр детальной информации о логах;
- Фильтрация логов по типу действия и дате;
- Просмотр смен курьеров и статистики по курьерам;
