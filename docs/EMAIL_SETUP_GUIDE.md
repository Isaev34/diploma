# Полная инструкция по настройке отправки Email

## 📧 Быстрый старт

### Шаг 1: Выберите провайдера

**Рекомендация для тестирования:** Gmail (проще всего настроить)

### Шаг 2: Настройте Gmail

#### Для Gmail нужно использовать пароль приложения (не обычный пароль!)

1. **Включите двухфакторную аутентификацию:**
   - Перейдите: https://myaccount.google.com/security
   - Включите "Двухэтапная аутентификация"

2. **Создайте пароль приложения:**
   - Перейдите: https://myaccount.google.com/apppasswords
   - Или: Безопасность → Двухэтапная аутентификация → Пароли приложений
   - Выберите "Почта" и "Другое устройство"
   - Введите название: "Django Shop"
   - Нажмите "Создать"
   - **Скопируйте 16-значный пароль** (он показывается только один раз!)

3. **Заполните настройки в `vkusvill_shop/settings.py`:**
```python
EMAIL_HOST_USER = 'ваш_email@gmail.com'  # Ваш Gmail адрес
EMAIL_HOST_PASSWORD = 'abcd efgh ijkl mnop'  # 16-значный пароль приложения (без пробелов или с пробелами - оба варианта работают)
DEFAULT_FROM_EMAIL = 'ваш_email@gmail.com'
```

### Шаг 3: Проверьте работу

1. Запустите сервер: `python manage.py runserver`
2. Оформите тестовый заказ
3. Проверьте почтовый ящик (включая папку "Спам")

## 🔧 Альтернативные провайдеры

### Yandex

1. Включите SMTP в настройках почты Yandex
2. В `settings.py` используйте:
```python
EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
EMAIL_HOST = 'smtp.yandex.ru'
EMAIL_PORT = 465
EMAIL_USE_TLS = False
EMAIL_USE_SSL = True
EMAIL_HOST_USER = 'ваш_email@yandex.ru'
EMAIL_HOST_PASSWORD = 'ваш_пароль'
DEFAULT_FROM_EMAIL = EMAIL_HOST_USER
```

### Mail.ru

1. Включите SMTP в настройках почты Mail.ru
2. В `settings.py` используйте:
```python
EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
EMAIL_HOST = 'smtp.mail.ru'
EMAIL_PORT = 465
EMAIL_USE_TLS = False
EMAIL_USE_SSL = True
EMAIL_HOST_USER = 'ваш_email@mail.ru'
EMAIL_HOST_PASSWORD = 'ваш_пароль'
DEFAULT_FROM_EMAIL = EMAIL_HOST_USER
```

## 🧪 Тестирование без реальной отправки

Если хотите просто проверить, что письма формируются правильно:

В `settings.py` используйте:
```python
EMAIL_BACKEND = 'django.core.mail.backends.console.EmailBackend'
```

Письма будут выводиться в консоль Django при запуске сервера.

---

## ⚠️ Важные замечания

1. **Для Gmail обязательно используйте пароль приложения**, не обычный пароль!
2. **Пароль приложения** - это 16 символов, может быть с пробелами или без
3. Если письма не приходят:
   - Проверьте папку "Спам"
   - Проверьте логи в консоли Django
   - Убедитесь, что email пользователя указан в профиле
4. **Не коммитьте пароли в Git!** Используйте переменные окружения для продакшена

---

## 🔒 Безопасность (для продакшена)

Для продакшена рекомендуется:

1. Использовать переменные окружения:
```python
import os
EMAIL_HOST_USER = os.environ.get('EMAIL_HOST_USER')
EMAIL_HOST_PASSWORD = os.environ.get('EMAIL_HOST_PASSWORD')
```

2. Или использовать специализированные сервисы:
   - SendGrid (бесплатно до 100 писем/день)
   - Mailgun (бесплатно до 5000 писем/месяц)
   - AWS SES (очень дешево)

## 🐛 Решение проблем

### Ошибка: "Authentication failed"
- Проверьте правильность пароля приложения (для Gmail)
- Убедитесь, что двухфакторная аутентификация включена

### Ошибка: "Connection refused"
- Проверьте настройки порта (587 для TLS, 465 для SSL)
- Проверьте, не блокирует ли файрвол соединение

### Письма не приходят
- Проверьте папку "Спам"
- Проверьте, что email пользователя указан в профиле
- Проверьте логи в консоли Django

### Письма приходят, но пустые
- Проверьте шаблоны в `templates/cart/emails/`
- Убедитесь, что все переменные в шаблоне корректны
