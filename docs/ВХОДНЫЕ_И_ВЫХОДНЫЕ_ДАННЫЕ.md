2.1.1. Входные данные предметной области

Входными данными будут являться: поле регистрации - имя пользователя (username), email, телефон, пароль, подтверждение пароля; поле авторизации - имя пользователя (username), пароль; поле изменения профиля - email, телефон; поле добавления адреса доставки - город, улица, дом, корпус, подъезд, этаж, квартира/офис, основной адрес; поле изменения адреса доставки - город, улица, дом, корпус, подъезд, этаж, квартира/офис, основной адрес; поле добавления карты - номер карты, тип карты (Visa, MasterCard, Мир), имя держателя карты, использование по умолчанию; поле фильтрации товаров - поисковый запрос (q или search), slug категории, тип сортировки (по цене по возрастанию/убыванию, по названию, по дате добавления), страница пагинации; поле сортировки товаров - тип сортировки (price_asc, price_desc, name, created_at); поле добавления в корзину - ID товара, количество; поле изменения корзины - ID позиции корзины, количество (при удалении - ID позиции корзины); поле создания заказа - адрес доставки (город, улица, дом, корпус, подъезд, этаж, квартира/офис), комментарий для курьера, комментарий для сборщика, способ оплаты (картой онлайн, картой при получении, наличными при получении), ID карты при оплате онлайн, количество списываемых бонусных баллов, флаг сохранения адреса в профиле; список товаров формируется из корзины (ID товара, количество); поле расчёта стоимости доставки (API) - город, улица, дом, корпус; поля администратора - создание пользователя - имя пользователя (username), имя, фамилия, email, телефон, пароль, роль (группа: Покупатель, Менеджер, Администратор, Курьер); поля администратора - изменение пользователя - имя пользователя, имя, фамилия, email, телефон, пароль, роль (группа), активность (is_active); поля менеджера - создание категории - название, slug, описание, изображение, признак активности; поля менеджера - изменение категории - название, slug, описание, изображение, признак активности; поля менеджера - создание товара - название, slug, описание, категория (ID), штрихкод, цена, процент скидки, изображение, количество на складе, признак активности; поля менеджера - изменение товара - название, slug, описание, категория (ID), штрихкод, цена, процент скидки, изображение, количество на складе, признак активности; поле изменения статуса заказа - статус (в ожидании, в сборке, собран, доставляется, доставлен, отменён); поле назначения сборщика на заказ (API) - ID заказа, ID пользователя-сборщика; поле назначения курьера на заказ (API) - ID заказа; поле обновления статуса заказа курьером (API) - ID заказа, новый статус (доставляется, доставлен); поле загрузки фотоотчёта доставки (API) - ID заказа, файл изображения (photo_report или delivery_photo); поле фильтрации логов активности - пользователь (ID), тип действия (action_type), начальная и конечная дата, страница, лимит; поле просмотра статистики - начальная дата, конечная дата; поле загрузки изображения товара - файл изображения, ID товара (через форму редактирования товара в админке).

2.1.2. Выходные данные предметной области

Выходными данными будут являться: результат авторизации; результат регистрации; вывод данных с поиска; вывод данных с фильтрации; получение товаров в окне «Каталог»; получение товаров в окне «Корзина»; результат формирования отчётов по доходам и продажам (дашборд аналитики за выбранный период); список всех пользователей; список всех товаров; список категорий; вывод данных пользователя в окне «Профиль» (имя пользователя, email, телефон, бонусные баллы, аватар, адреса доставки, карты); вывод информации о истории заказов в окне «История заказов»; список ролей (Покупатель, Менеджер, Администратор, Курьер); результат изменения статуса заказа; список логов пользователей; графики статистики (продажи по датам, топ товаров, заказы по статусам, сводные показатели); результат добавления товара в корзину; результат оформления заказа; детальная информация о товаре; результат создания/изменения товара; результат создания/изменения категории; результат создания/изменения пользователя; результат удаления товара; результат удаления категории; результат удаления пользователя; детальная информация о заказе (позиции, адрес доставки, способ оплаты, стоимость доставки, комментарии, статус, фотоотчёт); результат расчёта стоимости доставки по адресу (API); список заказов для сборщика (API); список заказов для курьера (API); результат загрузки фотоотчёта доставки (API); список смен курьеров (админка).
