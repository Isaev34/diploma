## Текст промпта

Сгенерируй **физическую модель (схему) базы данных** для курсовой работы. Проект: **интернет-магазин продуктов питания с доставкой** (Django). Нужна схема для «Рисунок 21 – Физическая модель данных».

**Требования к результату:**
- Формат: диаграмма в виде изображения, Mermaid, PlantUML, draw.io-описание или пошаговая инструкция для построения в draw.io/Visio.
- На схеме: сущности (таблицы), их атрибуты с типами данных, связи между таблицами (FK), ключи (PK, FK).
- Стиль: классическая ER-диаграмма или физическая схема БД (прямоугольники для таблиц, строки — поля с типами, связи — линии с подписями 1:N и т.д.).

---

**Состав БД (11 основных таблиц):**

### 1. users_user (Пользователь)
Наследуется от Django AbstractUser. Поля: id (PK), username, email, password, first_name, last_name, is_staff, is_active, is_superuser, date_joined, last_login; дополнительно: phone (VARCHAR 20), bonus_points (INT ≥ 0), avatar (файл), created_at, updated_at (datetime).

### 2. users_address (Адрес доставки)
id (PK), user_id (FK → users_user), city (VARCHAR 100), street (VARCHAR 200), house (VARCHAR 20), block (VARCHAR 20, null), entrance (VARCHAR 10, null), floor (INT, null), apartment (VARCHAR 20, null), latitude (DECIMAL 9,6, null), longitude (DECIMAL 9,6, null), is_default (BOOL), created_at, updated_at.

### 3. users_usercard (Карта пользователя)
id (PK), user_id (FK → users_user), card_type (VARCHAR 20: visa/mastercard/mir), last_four_digits (VARCHAR 4), cardholder_name (VARCHAR 100), is_default (BOOL), created_at.

### 4. users_useractivitylog (Лог активности)
id (PK), user_id (FK → users_user, null), action_type (VARCHAR 50), description (TEXT), ip_address (IP, null), user_agent (TEXT), request_path (VARCHAR 500), request_method (VARCHAR 10), status_code (INT, null), created_at.

### 5. catalog_category (Категория товаров)
id (PK), name (VARCHAR 100, unique), slug (VARCHAR 100, unique), description (TEXT), image (файл, null), is_active (BOOL), created_at, updated_at.

### 6. catalog_product (Товар)
id (PK), name (VARCHAR 200), slug (VARCHAR 200), description (TEXT), barcode (VARCHAR 50, unique, null), price (DECIMAL 10,2), discount_percent (INT 0–100), category_id (FK → catalog_category), image (файл, null), is_active (BOOL), in_stock (BOOL), stock_quantity (INT ≥ 0), created_at, updated_at. unique(name, category_id).

### 7. cart_order (Заказ)
id (PK), user_id (FK → users_user), courier_id (FK → users_user, null), picker_id (FK → users_user, null), status (VARCHAR 20), verification_code (VARCHAR 8, unique, null), payment_method (VARCHAR 20), payment_card_id (FK → users_usercard, null), delivery_city, delivery_street, delivery_house, delivery_block (null), delivery_entrance (null), delivery_floor (INT, null), delivery_apartment (null), delivery_lat, delivery_lng (DECIMAL 9,6, null), delivery_comment_courier (TEXT, null), delivery_comment_picker (TEXT, null), total_amount (DECIMAL 10,2), bonus_points_used (INT), bonus_points_earned (INT), photo_report (файл, null), courier_fee (DECIMAL 10,2), delivery_fee (DECIMAL 10,2), created_at, updated_at.

### 8. cart_orderitem (Позиция заказа)
id (PK), order_id (FK → cart_order), product_id (FK → catalog_product, null), quantity (INT ≥ 1), expected_barcode (VARCHAR 50, null), is_replacement (BOOL), replaced_item_id (FK → cart_orderitem, null), replacement_comment (VARCHAR 255, null), custom_product_name (VARCHAR 255, null), price_at_purchase (DECIMAL 10,2), is_scanned (BOOL). unique(order_id, product_id).

### 9. cart_cart (Корзина)
id (PK), user_id (FK → users_user, null), session_key (VARCHAR 40, null), created_at, updated_at.

### 10. cart_cartitem (Позиция корзины)
id (PK), cart_id (FK → cart_cart), product_id (FK → catalog_product), quantity (INT ≥ 1), created_at, updated_at. unique(cart_id, product_id).

### 11. shifts_shift (Смена курьера)
id (PK), courier_id (FK → users_user), started_at (datetime, null), ended_at (datetime, null), is_confirmed (BOOL), status (VARCHAR 20: pending/started/ended), orders_count (INT ≥ 0), earnings (DECIMAL 10,2), online_minutes (INT ≥ 0).

---

**Связи (кратко):**
- User 1:N Address, User 1:N UserCard, User 1:N UserActivityLog, User 1:N Order (как заказчик), User 1:N Cart, User 1:N Shift (как курьер).
- Order N:1 User (courier, picker — опционально), Order N:1 UserCard (опционально); Order 1:N OrderItem.
- OrderItem N:1 Order, N:1 Product (опционально), self-reference replaced_item_id (замена).
- Category 1:N Product.
- Cart 1:N CartItem; CartItem N:1 Product.
- Shift N:1 User (courier).
