2.1.3. Требование к проекту

Разработать реляционную базу данных на основе СУБД PostgreSQL для организации хранения и управления информацией о товарах, пользователях, заказах, адресах доставки и справочных данных интернет-магазина продуктов питания.

Спроектировать и реализовать веб-приложение, обеспечивающее клиентам возможность просмотра каталога товаров (продукты питания по категориям), использования функций поиска, применения сортировки и фильтрации по категории, оформления заказа с указанием адреса доставки и способа оплаты, просмотра истории заказов и редактирования личных данных и адресов доставки в профиле.

Пользовательский интерфейс веб-сайта должен обладать интуитивной понятностью и простотой использования для обычных пользователей. Необходимо обеспечить удобную навигацию между разделами сайта с помощью понятных элементов управления.

Для получения доступа к полному функционалу системы в соответствии с назначенной ролью пользователь должен пройти процедуру авторизации. Новые пользователи могут зарегистрироваться в системе, указав следующие данные: имя пользователя (username), email, телефон и пароль.

Управление пользователями и просмотр системных логов доступны только администраторам. Менеджеры имеют права на управление товарами и категориями, изменение статусов заказов и просмотр статистики продаж.

Веб-сайт интернет-магазина продуктов с доставкой должен иметь следующие функциональные возможности:

Общие требования (для всех пользователей)
- Требование к авторизации;
- Требование к регистрации;
- Выход из аккаунта;
- Просмотр товаров в каталоге (по категориям);
- Требование к фильтрации товаров по категории;
- Требование к сортировке товаров (по цене, по названию, по дате добавления);
- Требование к поиску товара;
- Просмотр страницы товара с подробной информацией;

Требования для авторизованных клиентов (роль «Покупатель»)
- Просмотр товаров в корзине;
- Требование к добавлению товаров в корзину;
- Требование к изменению количества товара в корзине;
- Требование к удалению товаров из корзины;
- Возможность оформления заказа с указанием адреса доставки, способа оплаты и комментариев для курьера и сборщика;
- Требование к списанию бонусных баллов при оформлении заказа;
- Просмотр стоимости доставки по введённому адресу (расчёт по расстоянию);
- Сохранение адресов доставки в профиле;
- Добавление и управление привязанными картами (для оплаты онлайн);
- Просмотр истории заказов;
- Просмотр детальной информации о заказе (статус, позиции, адрес, способ оплаты);
- Требование к изменению данных профиля (email, телефон);
- Просмотр и управление списком адресов доставки;

Требования для менеджеров (роль «Менеджер»)
Управление заказами
- Требование к просмотру списка заказов пользователей;
- Требование к обновлению статусов заказов (в ожидании, в сборке, собран, доставляется, доставлен, отменён);
- Назначение сборщика на заказ (через админку или API);

Управление товарами
- Требование к просмотру списка товаров;
- Требование к добавлению данных о товарах (название, описание, категория, штрихкод, цена, скидка, изображение, остаток на складе);
- Требование к изменению данных о товарах;
- Требование к удалению данных о товарах;
- Требование к загрузке изображений для товаров;

Управление категориями
- Требование к просмотру списка категорий;
- Требование к добавлению данных о категориях (название, slug, описание, изображение);
- Требование к изменению данных о категориях;
- Требование к удалению данных о категориях;

Отчёты и статистика
- Требование к просмотру отчётов по продажам;
- Требование к просмотру отчётов по доходам;
- Требование к формированию отчётов за выбранный период (дашборд аналитики: продажи по датам, топ товаров, заказы по статусам);

Требования для администраторов (роль «Администратор»)
- Требование к просмотру списка пользователей;
- Требование к добавлению данных о пользователях (имя пользователя, имя, фамилия, email, телефон, пароль, роль);
- Требование к изменению данных о пользователях (имя пользователя, имя, фамилия, email, телефон, пароль, роль, активность);
- Требование к удалению данных о пользователях;
- Требование к просмотру логов активности пользователей;
- Требование к фильтрации логов по типу действия и дате;
- Просмотр смен курьеров и статистики по курьерам (при наличии роли курьера в системе);

Примечание. Работа курьеров и сборщиков заказов в проекте обеспечивается через мобильные приложения и API платформы; отдельный перечень требований к интерфейсу приложений в данном разделе не приводится.
